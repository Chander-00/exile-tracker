# Dockerization Design

**Date**: 2026-03-03
**Status**: Approved

## Context

Exile-tracker is a Go monolith with 4 integrated components (HTTP API, Web UI, SSH TUI, background fetcher) sharing a single SQLite database. The goal is to dockerize it for deployment on a cheap VPS, with Nginx as a reverse proxy and Cloudflare for TLS.

## Architecture

```
Internet
  |
  +-- Port 80/443 (HTTP/HTTPS) --> [Nginx container] --> [exile-tracker container :3000]
  |
  +-- Port 2222 (SSH) -----------> [exile-tracker container :2222] (direct)
```

- Single container for the Go monolith (all 4 components)
- Nginx container as reverse proxy with Cloudflare Origin Certificate for TLS
- Docker volumes for persistent data (SQLite DB, SSH host key)
- Path of Building (user's fork) cloned at build time

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Container strategy | Single container | Matches monolith architecture, SQLite doesn't work across containers |
| PoB handling | Clone at build time | Self-contained image, update by rebuilding |
| PoB source | User's fork (github.com/Chander-00/PathOfBuilding) | Has custom modifications for headless mode and sorted Lua tables |
| SSH host key | Docker volume | Persists across restarts, auto-generated by wish library |
| Reverse proxy | Nginx in compose | User wants to learn Nginx, easy to extend for future apps |
| TLS | Cloudflare Origin Certificate | Free, 15-year validity, no certbot/renewal needed |
| Deploy workflow | git pull + docker compose up --build | Simple, rebuilds PoB fresh each time |

## Files to Create

### 1. Dockerfile (multi-stage)

**Stage 1 - Builder:**
- Base: `golang:1.25-bookworm`
- Install `templ` CLI
- Copy go.mod/go.sum, download dependencies
- Copy source code, run `templ generate`
- Build static Go binary with CGO enabled (required for SQLite)

**Stage 2 - Runtime:**
- Base: `debian:bookworm-slim` (glibc needed for CGO/SQLite, plus luajit)
- Install: `luajit`, `git`, `ca-certificates`
- Clone PoB fork from GitHub
- Copy Go binary from stage 1
- Copy migrations folder
- Expose ports 3000 and 2222
- Entrypoint: run the binary

### 2. docker-compose.yml

```yaml
services:
  exile-tracker:
    build: .
    env_file: .env
    volumes:
      - sqlite-data:/app/data
      - ssh-keys:/app/.ssh
    ports:
      - "2222:2222"
    restart: unless-stopped

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/certs:/etc/nginx/certs:ro
    depends_on:
      - exile-tracker
    restart: unless-stopped

volumes:
  sqlite-data:
  ssh-keys:
```

### 3. nginx/nginx.conf

```nginx
events {
    worker_connections 1024;
}

http {
    server {
        listen 80;
        server_name yourdomain.com;
        return 301 https://$host$request_uri;
    }

    server {
        listen 443 ssl;
        server_name yourdomain.com;

        ssl_certificate     /etc/nginx/certs/origin.pem;
        ssl_certificate_key /etc/nginx/certs/origin-key.pem;

        location / {
            proxy_pass http://exile-tracker:3000;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
}
```

### 4. .env.example

Template of required environment variables for production deployment.

### 5. .dockerignore

Exclude unnecessary files from build context (bin/, data.db, .ssh/, .env, etc.).

## Application Code Changes

**None.** The Go code stays untouched. Only environment variable values differ in production:

- `DB_PATH=/app/data/data.db` (inside the mounted volume)
- `POB_ROOT=/app/PathOfBuilding` (cloned inside the container)
- `SSH_HOST_KEY_PATH=.ssh/exile_tracker_ed25519` (already the default)

## Deployment Workflow

### First time:
1. SSH into VPS
2. Install Docker (`curl -fsSL https://get.docker.com | sh`)
3. Clone repo
4. Create `.env` from `.env.example`
5. Place Cloudflare origin cert in `nginx/certs/`
6. `docker compose up -d --build`

### Updates (app or PoB):
1. `git pull`
2. `docker compose up -d --build`

## Future Extensibility

When adding a second app to the same VPS:
- Add new service to docker-compose.yml
- Add new server block in nginx.conf (different subdomain)
- Nginx routes traffic to the correct container
